Discovery

Generally, I’ll start a project out with a discovery phase, where I work on understanding the project parameters, the business needs and goals, and any competitors.

View an example from my work

User Research

Next, I’ll move into user research, where I try to understand who the users are, and how we can bring them value with the design that we create. To do this, I’ll do things like user interviews, and then use the information I gather to build personas, user journey maps, empathy maps, scenario maps, and storyboards, and then present my findings in an insight report. This is also a good place to stop and work with the client to create a plan for an MVP, so we know what to prioritize as we begin designing.

Information Architecture

For the information architecture phase of the UX process, I will take my understanding of the target audience, and create user stories. These user stories then help me create a card sorting task, which I will have the users organize. The results of the card sorting will then inform my site map. I’ll also create user flows, in order to make sure that I accommodate the users’ main goals.

Design

Next, I’ll draw some quick, high level sketches as a form of brainstorming. I’ll share my sketches with my team to get some feedback. Then I’ll create wireframes using an application like Sketch. At this point, I’m not worrying about stylistic options, but rather focusing on the structure and functionality of the product. Again, I’ll get feedback from my team on the wireframes. At this phase, I’ll also start a style guide that I can update and modify as the design solidifies. Finally, I’ll move on to creating prototypes using Sketch and something like Invision.

View an example from my work

User Testing

Using the prototypes that I created, next I’ll do some user testing, to get the product in front of users and discover what works and and what doesn’t. Then I will adjust the designs based on the feedback I received.

View an example from my work

Handoff

Once I’ve made any revisions based on the results of the testing, I’ll pass the design off to the developers that are collaborating with me. I’ll work with the developers to make sure that the designs make sense and answer any questions they might have.
