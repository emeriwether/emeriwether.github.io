• Discovery

    Generally, I'll start a project out with a discovery phase, where I work on understanding the project parameters, the business needs and goals, and any competitors the project might have.

  • User Research

    Next, I'll move into user research, where I try to understand who the users are, and how we can bring them value with the design that we create. To do this, I'll conduct user interviews and surveys, and then use the information I gather to help me, my team, and my client connect with the users. After the deliverables are complete, I'll usually present my findings in an insight report. I find that this is also a good place to work with the client to create a plan for an MVP, so that we know what to prioritize as we begin designing.

  • Information Architecture

    To create the information architecture of the product, I’ll use my understanding of the target audience to create user stories, do card sorting tasks with users, and create a sitemap. I’ll also usually create user flows, in order to make sure that I accommodate the users’ main goals.

  • Design

    Next, I’ll move into the design phase. I’ll usually start with sketches, and then I’ll use programs such as Sketch, Photoshop, and Invision to create wireframes and prototypes. I’ll also create a style guide early on in the design phase, and then adjust it as the design changes and formalizes.

  • User Testing

    Using the prototypes that I created, next I’ll do some user testing, to get the product in front of users and discover what works and and what doesn’t. Then I'll make revisions on the design based on the results of the testing.

  • Hand-off

    Finally, I’ll pass the design off to the developers that are collaborating with me. I like to work as closely as possible with the developers to make sure that I've given them everything they need, that the designs make sense, and to answer any questions they might have.
